# Implementation Plan: Local Transformers Provider for ReAct Experiments

**Branch**: `001-local-transformers-provider` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-local-transformers-provider/spec.md`

## Summary

Add a `TransformersProvider` that loads Qwen/Qwen3-8B locally via the `transformers`
library and plugs into the existing `LLMProvider` Strategy Pattern. Extends `LLMConfig`
with two optional fields (`device_map`, `torch_dtype`), registers the new provider in
`LLMProviderFactory`, and adds a new Hydra config for local ReAct evaluation.
No changes to the ReAct evaluator or task planner are required.

## Technical Context

**Language/Version**: Python 3.10+ (project targets 3.8–3.12; `transformers>=4.51.0` requires 3.8+)
**Primary Dependencies**: `transformers>=4.51.0`, `torch>=2.0.0`, `accelerate>=0.26.0` (new); `hydra-core`, `openai`, `pytest` (existing)
**Storage**: Files — HuggingFace model cache (`~/.cache/huggingface/`), JSON evaluation outputs
**Testing**: pytest (existing `tests/` suite)
**Target Platform**: Linux server with CUDA GPU (primary); CPU-only supported
**Project Type**: CLI / research evaluation framework (library)
**Performance Goals**: Model loads once per evaluation run; per-task inference speed is hardware-dependent
**Constraints**: Must implement `LLMProvider` ABC (`chat_completion()`, `select_action()`); must be config-selectable via `provider: "transformers"`; output schema must match existing providers
**Scale/Scope**: Single researcher, single machine, serial per-task inference

## Notes

- All tests mock `AutoModelForCausalLM.from_pretrained` and `AutoTokenizer.from_pretrained`
  — no actual model download is required during testing.
- The `torch_dtype` validation (Group 3, `test_provider_init_invalid_torch_dtype_raises`)
  must fire BEFORE calling `from_pretrained` to avoid partial downloads on bad config.
- After each "go" cycle, run `pytest tests/ -v` to confirm no regressions in the
  existing provider tests (`test_llm_providers.py`, `test_react_planner.py`, etc.).
