# Implementation Plan: Instance-Aware Object Tracking in ReAct Loop

**Branch**: `006-instance-object-tracking` | **Date**: 2026-03-10 | **Spec**: [spec.md](spec.md)

## Summary

Expose object instance labels (`Cup_1`, `Egg_2`) in pick and put success observations so the LLM can refer back to specific objects across multi-step transformation episodes. Update the heat few-shot example to demonstrate instance-specific return navigation (`find Egg_1` instead of `find a egg`). No new classes, no new config keys — two return-value changes in `ThorConnector` and one updated prompt template.

## Implementation Notes

### Change 1 — `pick()` success observation (`thor_connector.py:502-503`)

**Before**:
```python
if self.last_event.metadata['lastActionSuccess']:
    ret_msg = ''
```

**After**:
```python
if self.last_event.metadata['lastActionSuccess']:
    ret_msg = f"Picked up {self.readable_id(obj_id)}."
```

`obj_id` is already in scope (resolved earlier in `pick()`). `readable_id()` performs a single dict lookup against `_obj_registry`.

---

### Change 2 — `put()` success observation (`thor_connector.py:591-592`)

**Before**:
```python
ret_msg = ''
halt = True
```

**After**:
```python
ret_msg = f"Put {self.readable_id(holding_obj_id)} in {self.readable_id(recep_id)}."
halt = True
```

Both `holding_obj_id` (line 516) and `recep_id` (loop variable) are already in scope at the success branch.

---

### Change 3 — Few-shot "hot egg" example (`react_few_shot_examples.txt`)

The existing hot egg example uses `find a egg` when retrieving the egg from the microwave. After the change, observations will read `"Picked up Egg_1."` and `"Put Egg_1 in Microwave_1."`, so the example must be updated to:
- Show the new observation format in existing steps
- Replace the post-heating `find a egg` + `pick up the egg` with `find Egg_1` + `pick up the Egg_1`

No other examples need to change; few-shot generalisation covers cool and clean cases.

---

### Test plan

**`tests/test_thor_connector_observations.py`** — new file:

1. `test_pick_success_observation_includes_instance_label` — mock a successful pick; assert return value matches `"Picked up <Type>_<N>."` pattern
2. `test_pick_failure_observation_unchanged` — assert failure messages are not modified
3. `test_put_success_observation_includes_object_and_receptacle_labels` — mock a successful put; assert `"Put <Type>_<N> in <Type>_<M>."` format
4. `test_put_failure_observation_unchanged` — assert failure messages unchanged
5. `test_readable_id_used_for_both_object_and_receptacle` — verify the exact label values match the registry entries for those objectIds

Existing tests (`test_instance_actions.py`, 53 tests) must all pass after the changes.
